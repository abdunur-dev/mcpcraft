## What is MCP?

The **Model Context Protocol (MCP)** is an open standard that lets AI apps (Claude, Cursor, VS Code Copilot) call functions and read data from your server — the same way HTTP lets browsers talk to APIs.

An MCP server exposes **tools** (functions an LLM can execute) and **resources** (data an LLM can read). Without MCP, you'd need to build custom integrations for every AI app. With MCP, you write it once and any compatible client can use it.

<Callout type="info">
  MCP is to AI what HTTP is to the web — a universal protocol that decouples clients from servers. mcpcraft-sdk is to MCP what Express is to HTTP.
</Callout>

## Why mcpcraft-sdk?

The official `@modelcontextprotocol/sdk` requires ~100 lines of boilerplate for a simple server. mcpcraft-sdk distills that down to ~10 lines — with automatic schema generation, type inference, and input validation.

```typescript
import { createServer, tool } from "mcpcraft-sdk"

const server = createServer({ name: "hello-world" })

server.add(tool({
  name: "greet",
  description: "Greets a user by name",
  input: {
    name: { type: "string", description: "The user's name" }
  },
  run: async ({ name }) => {
    return { message: `Hello, ${name}!` }
  }
}))

server.start()
```

<Callout type="success">
  That's it. 12 lines and your MCP server is ready to connect to any MCP-compatible client.
</Callout>

## Features

### Zero Boilerplate
Define tools with plain objects. No JSON-RPC handlers, no transport setup, no manual schema definitions.

### Full Type Safety
Handler parameters are typed automatically from your input schema — full autocomplete, zero casts.

### Built-in Validation
Every tool input is validated with Zod before reaching your handler. Invalid calls get structured error responses, not crashes.

### CLI Scaffolding
Scaffold a complete project in one command: `npx mcpcraft-sdk init`

### Tools & Resources
Register executable tools (for LLM actions) and read-only resources (for context) with the same `.add()` API.

### Production Ready
Built on top of the official `@modelcontextprotocol/sdk`. Fully spec-compliant, works with Claude Desktop, VS Code Copilot, Cursor, and all MCP hosts.

## Quick Comparison

| Task | Raw SDK | mcpcraft-sdk |
|------|---------|-------------|
| Server boilerplate | 30+ lines | 3 lines |
| Tool registration | Manual schema + handlers | `server.add(tool({...}))` |
| Input validation | Manual type checks | Automatic Zod |
| Transport setup | Manual stdio config | Built-in `.start()` |
