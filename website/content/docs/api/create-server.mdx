# createServer

Initialize and configure a new Model Context Protocol (MCP) server instance.

```typescript
import { createServer } from "mcpkit"

const server = createServer({
  name: "my-server",       // Required. Used to identify the server.
  version: "1.0.0",        // Optional. Defaults to "0.1.0".
  description: "My server" // Optional. A short description of the server.
})
```

---

## Methods

### `server.add(item)`
Registers a tool or a resource with the server. Returns the server instance for easy chaining.

- **Arguments**:
  - `item`: A `Tool` or `Resource` object generated by the `tool()` or `resource()` helpers.
- **Returns**: `McpServerInstance` (chainable).

```typescript
server
  .add(tool({ name: "tool_a", ... }))
  .add(resource({ name: "resource_b", ... }))
```

### `server.start()`
Launches the server on standard input/output (`stdio`) transport. It configures the official JSON-RPC message handlers under the hood.

- **Returns**: `Promise<void>`.

```typescript
await server.start()
```

### `server.server`
Exposes the underlying official `Server` instance from `@modelcontextprotocol/sdk/server/index.js` for developers needing low-level access to the protocol capabilities, notifications, or custom transports.
